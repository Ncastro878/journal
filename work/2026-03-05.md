# Replace GPT Vision with AWS Textract for RO Text Extraction

**Project:** eCustom-Solutions/jobs-service
**PRs:** https://github.com/eCustom-Solutions/jobs-service/pull/718

Refactored the generateROs HubSpot job to replace OpenAI GPT-5 vision-based OCR with AWS Textract. The existing GPT approach was unreliable with low-quality phone photos of repair order receipts (crumpled papers, poor lighting, stains), often hallucinating text or failing to extract content accurately.

## Key Changes
- Created `src/services/integrations/textract.js` with synchronous `DetectDocumentText` API
- Replaced GPT OCR calls in `src/helpers/ocr.js` with Textract
- Added confidence scoring - pages below 60% confidence flagged as low quality (same format as GPT fallback)
- Switched region from us-east-2 to us-east-1 for 10 TPS limit (vs ~1-3 TPS)
- Increased concurrency from 8 to 10 to match TPS limit
- Added temp file cleanup to delete `/tmp` images after processing

## Research Findings
- Textract sync API: single page, ≤10MB, immediate response
- Textract async API: multi-page PDFs up to 500MB/3000 pages, requires S3
- Chose sync API as drop-in replacement - minimal code changes
- Smartphone photos typically 2-5MB, well under 10MB limit
- Cost: ~$0.0015/page (Textract) vs ~$0.002-0.003/image (GPT-4o), GPT-5 likely more

## Production Verification
- Deployed and tested with 12 docs processed in ~3.3 minutes
- ~16 seconds per doc end-to-end (includes HubSpot API calls, AI summarization)

## AWS Setup Required
- IAM permission: `textract:DetectDocumentText`
- Monitor costs in AWS Cost Explorer → filter by "Amazon Textract"

## Notes
- Textract won't hallucinate - returns confidence scores instead
- Could further optimize with image preprocessing (grayscale, sharpen, binarize) - function exists but unused
- Future consideration: async Textract for processing entire PDFs at once
